Introduction to Conditional Statements

Computational Thinking Principle

Conditionals enable decision-making in programs using logical conditions and branching, allowing your code to respond differently to different situations.

Without Conditionals
With Conditionals

Conditional Advantages

  • Dynamic behavior: Programs can adapt to different inputs
  • Error handling: Manage different scenarios gracefully
  • Complex logic: Build sophisticated decision trees

Real-Life Examples

  • Traffic lights: Change based on sensor inputs
  • Thermostats: Adjust temperature based on readings
  • Login systems: Grant access only with valid credentials

If Statement - Simple Decisions

The if statement executes code only when a condition is True. It's Python's most basic conditional structure.

Syntax

if condition:
    # code to execute if condition is True
# code that always runs
Common Mistakes
Proper Implementation

Best Practices

  • Keep conditions simple and readable
  • Use descriptive variable names in conditions
  • Avoid complex logic in the condition itself
  • Use parentheses only when needed for operator precedence

If-Else - Binary Choices

The if-else structure lets you choose between exactly two paths based on a condition.

Syntax

if condition:
    # code when true
else:
    # code when false
# code that always runs
Problematic Patterns
Clean Implementation

When to Use

  • When you have exactly two mutually exclusive paths
  • For simple true/false decisions
  • When the "else" case is clearly defined

If-Elif-Else - Multiple Conditions

The if-elif-else chain handles multiple possible conditions in order.

Syntax

if condition1:
    # code for condition1
elif condition2:
    # code for condition2
elif condition3:
    # code for condition3
else:
    # default case
# code that always runs
Common Issues
Proper Structure

Best Practices

  • Order conditions from most specific to least specific
  • Use mutually exclusive conditions when possible
  • Include a final else clause as a catch-all
  • Limit to 3-5 elifs - consider a dictionary or match-case for more

Nested Conditionals

Nested conditionals place if statements inside other if statements for complex decision trees.

Syntax

if condition1:
    if condition2:
        # both true
    else:
        # condition1 true, condition2 false
else:
    # condition1 false
Poor Nesting
Clean Nesting

When to Nest

  • When conditions are truly dependent (one must be true to check the next)
  • For hierarchical decision making
  • When the logic can't be expressed with simple and/or operators

Need Help?

If you get stuck, try breaking each problem into smaller steps. For example, in the Shopping Discount Calculator:

  1. First check if purchase >= 200
  2. Then check the next range (100-200) with elif
  3. Then check 50-100
  4. Finally handle the "else" case for under 50
Remember to test your code with different values to make sure all conditions work properly!
